해서는 안될 Javascript Code

2019-09-06

Javascript의 중요성이 날로 커지고 있다. 클라이언트 사이드뿐만 아니라 서버 사이드, 심지어 데스크탑 애플리케이션 개발이 가능해졌다.
필자는 Javascript 언어 하나만 잘 마스터해두면 일할 기회가 반드시 찾아올 것이라는 확신이 들기 시작했고 도전해보기로 결심했다.

Javascript 언어의 사양만 마스터했다고 만족해서는 안된다. 날마다 성장하고 변화하는 프레임워크 트랜드를 따라잡아야 하고 수많은 라이브러리의 사용법을 숙지해야한다. 그래도 그 중심에는 항상 언어가 자리 잡고 있다. 언어를 잘 이해하면 프레임워크, 라이브러리 내부를 살펴볼 수 있다. 필자가 아는 유명한 작품은 소스가 모두 오픈돼 있어 사용법은 물론 설계/철학, 그리고 Javascript Guru의 코드 작성 비결을 전수받을 수도 있다. Nothing is better than this!

이 시리즈는 임베디드 시스템 환경에서만 개발해오던 필자가 Javascript를 다루면서 겪은 경험(주로 삽질)을 공유하기 위해 작성했다.
삽질은 책으로 출간이 잘 안된다. 사람 수만큼 경우의 수가 다양하기 때문에 사례집으로 엮기도, 공감하기도 어렵기 때문이 아닐까.
필자처럼 컴파일러 언어를 중심으로 경험을 축적해온 사람들에게는 낯설기만한 Javascript라는 새로운 환경에 적응하는데 약간이나마 도움이 됐으면 좋겠다.

필자는 Javascript를 경험하기 시작한지 얼마 안 된 Newbie에 가깝다.
Javascript Guru가 보면 한심하게 느낄지 모르겠다. 넓은 아량으로 ‘성장하고 있구나’라고 봐주셨으면 좋겠다.
